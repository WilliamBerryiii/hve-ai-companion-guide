Part II builds on the engineering foundations from Part I, guiding you through each GitHub Copilot chat mode and advanced workflow patterns. You'll learn to apply RPI framework variations across discovery, planning, and implementation scenarios, achieving proficiency in AI-assisted development through practical, evidence-based exercises.

## What You'll Learn in Part II

Part II covers RPI framework implementation through Chapter 11, introducing you to all five GitHub Copilot chat modes with practical applications:

* **Ask Mode fundamentals** for quick questions and exploration
* **Task Researcher Mode** for systematic evidence-based research with mandatory citations
* **Task Planner Mode** for strategic planning with milestone tracking
* **Edit Mode** for precision implementation and targeted refactoring
* **Agent Mode** for autonomous execution with tool integration
* **ADR Creator Mode** for documenting architectural decisions
* **Advanced workflows** including the Beads workflow for systematic task breakdown

Each chapter introduces a chat mode with clear learning objectives, hands-on exercises, and real-world scenarios. By Chapter 11, you'll have learned advanced workflow patterns that serve as foundations for the custom agents and meta-prompting techniques introduced in Part III.

## Prerequisites

Before starting Part II, you should have completed Part I (Chapters 1-4), which covers:

* Engineering-ready development practices
* Local development environment setup
* RPI framework fundamentals (Request-Proof-Implement)
* Your first RPI workflow implementation
